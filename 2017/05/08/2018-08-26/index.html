git常用知识点记录

发表于 2017-05-08 | 分类于 前端

前言

git常用知识点记录，在团队开发中相信大家都会用到git，下面是一些指令的记录

基础指令

  • git init 初始化
  • git add 添加文件到暂存区
  • git commit 提交暂存区文件到版本库
  • git status 查看当前仓库状况
  • git diff 查看修改
  • git clone 拉取项目

版本回退

git reset –hard HEAD^

  • 回退到上一个版本

git reset –hard xxxx

  • 恢复到指定版本 xxxx为版本号，可通过git log ，git reflog等查询到记录

文件撤销修改

git checkout – file

  • 文件没提交到暂存区，恢复到和版本库一样的状态
  • 文件已提交到暂存区，恢复到刚提交到暂存区的状态

git reset HEAD file

  • 把提交到暂存区的修改撤销，退回到工作区

撤销修改的方法：

  1. 没有提交到暂存区的文件直接使用git checkout – file 恢复到和版本库一样的状态
  2. 已提交到暂存区的文件使用git reset HEAD file 退回到工作区，再使用git checkout – file 恢复和版本库一样的状态再使用git checkout – file 恢复到版本库一样的状态

删除文件

直接在使用文件管理器删除文件，或者用命令行删除。
删除完后使用git rm file指令确认从版本库中删除文件，再使用commit到版本库。如果是误删文件，使用git checkout – file 恢复

创建与合并分支

创建xxx分支
git branch xxx

切换到xxx分支
git checkout xxx

删除xxx分支
git branch -d xxx

强制删除xxx分支
git branch -D xxx

解决冲突

手动解决冲突，修改冲突

分支管理

使用–no-ff模式合并分支，git默认使用fast forward 模式进行合并，这样导致无法查看分支信息
故我们可以禁用fast forward模式，也就是说我们在合并分支的时候使用加上–no-ff选项，相当于一次提交
git merge –no-ff -m’合并dev’ dev

开发功能

接到一个新功能时，我们首先可以创建一个功能分支
使用git checkout feature -b 这是句指令相当于以下两步

  • git branch feature
  • git checkout feature

在此分支上开发功能开发完毕后在切换到dev分支（个人分支）进行合并操作，最后推送到远程的dev分支（个人分支）

修改bug

git stash

  • 保存当前工作状态，使用git stash list可以查看当前保存的工作状态

git stash apply

  • 恢复工作状态，恢复后不会删除工作状态需要使用git stash drop 删除

git stash drop

  • 删除工作状态

git stash pop

  • 恢复工作状态，并删除保存的工作状态

保存当前工作状态使用git stash,这个指令可以把当前的工作存放起来，当你在开发一个新需求开发一半的时候需要修改版本的bug，这时候你还没有开发完，所以你需要保存好当前开发现场，在新建一个临时分支，在临时分支上修改bug，修改完之后合并到dev分支上

多人协作

拉去远程仓库到本地，url为仓库地址
git clone url

查看远程仓库信息
git remote/git remote -v

创建远程分支dev
git checkout -b dev origin/dev

提交到远程仓库origin 的 xxx分支
git push origin xxx

本地dev分支关联远程dev分支，设置好才能pull
git –set-upstream-to=origin/dev dev

获取远程仓库更新
git pull
